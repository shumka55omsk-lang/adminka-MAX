# MAX Admin MVP v23 — Soft Consent + Mini App Audience

Версия v23 убирает большой блок согласия из мини-приложения, оставляет короткую подпись под кнопкой “Заказать замер” и сохраняет переходы для статистики/Яндекс-аудиторий.

## Что нового

- `/api/miniapp-visit` — сохраняет каждое открытие мини-приложения.
- `max_miniapp_visits` — журнал всех переходов и событий.
- `max_miniapp_audience` — агрегированная аудитория посетителей.
- `visitor_id` и `visit_id` — связка перехода, заявки и UTM.
- Поддержка Яндекс Метрики через `YANDEX_METRIKA_COUNTER_ID`.
- Цели Метрики: `miniapp_open` и `miniapp_lead`.
- В админке появился блок “Аудитория / Яндекс Директ”.
- `/api/yandex-audience-export` — экспорт CSV телефонов/хешей для Яндекс Аудиторий.
- В CSV попадают только заявки с телефоном и согласием `ad_consent=true`.

## Важно про Яндекс Директ

Все переходы в мини-приложение сохраняются в Supabase. Но для запуска рекламы именно на этих людей в Яндексе есть два рабочих сценария:

1. **Ретаргетинг через Яндекс Метрику** — добавить `YANDEX_METRIKA_COUNTER_ID`, создать цели `miniapp_open` и `miniapp_lead`, затем использовать эти цели/сегменты в Директе.
2. **CRM-аудитория** — выгружать телефоны пользователей, которые оставили заявку и дали согласие на рекламу, через CSV из админки.

MAX user_id сам по себе нельзя напрямую загрузить в Яндекс Директ как рекламный идентификатор. Поэтому посетители без телефона сохраняются для статистики, но для Яндекс Аудиторий нужны телефон/email или сегмент Метрики.

## Обновление Supabase

Выполнить в Supabase SQL Editor:

- полностью `supabase/schema.sql`, или
- только `supabase/hotfix-miniapp-audience.sql`.

## Переменные Vercel

```env
YANDEX_METRIKA_COUNTER_ID=12345678
PRIVACY_POLICY_URL=https://ваш-сайт.рф/privacy
MINIAPP_CONSENT_TEXT=Согласен(на) на обработку персональных данных для расчёта и связи по заявке.
```

После изменения переменных сделать Redeploy.

## Проверка

1. Открыть `/api/version` — должна быть версия `v23-soft-consent`.
2. Открыть `/miniapp?utm_campaign=test&utm_content=post_1`.
3. В Supabase проверить `max_miniapp_visits`.
4. Отправить заявку с согласием на рекламу.
5. В админке открыть блок “Аудитория / Яндекс Директ”.
6. Скачать CSV телефонов или SHA-256.

## Совместимость Vercel Hobby

В папке `api` должен быть только один файл:

```text
api/[...route].js
```

Файла `vercel.json` в корне быть не должно.


## v23 Soft Consent

В мини-приложении убран большой отдельный блок согласия. Теперь под кнопкой “Заказать замер” показывается короткая подпись: “Нажимая кнопку, вы соглашаетесь с обработкой данных для связи по заявке”.

Галочка для рекламы стала мягкой и необязательной: “Хочу получать акции и скидки по мягким окнам”. Если она не отмечена, заявка всё равно отправляется, визит сохраняется для статистики, но контакт не помечается как согласный на рекламные предложения.

Supabase заново обновлять не обязательно, если уже применён v22/hotfix-miniapp-audience.sql.
